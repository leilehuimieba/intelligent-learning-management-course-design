# Proposal - governance-bootstrap

## 标题
建立项目执行规范、AI 协作规范与 change 工作区底盘

## 背景
当前仓库已有 `courseapp` 主线和多份答辩文档，但缺少：

1. 唯一执行入口
2. 阶段口径
3. change 工作区
4. AGENTS 协作规则

同时仓库里还保留旧模块，若不正式写明主线，会导致后续 AI 或人推进时口径混乱。

## 目标
1. 明确 `courseapp` 为唯一执行主线
2. 明确旧模块为历史遗留，只读
3. 建立 docs 主入口
4. 建立阶段文档
5. 建立 changes 工作区
6. 建立 AGENTS 规则

## 非目标
1. 不修改 `courseapp` 业务代码
2. 不重构旧 Kotlin/Compose 多模块
3. 不增加新的业务功能

## 范围
本 change 仅覆盖：

1. docs 规范文件
2. AGENTS 协作规则
3. 当前阶段与当前活跃 change 定义

## 风险
1. 规范建立后若不持续使用，会再次失效
2. 若后续任务绕过 change 工作区，主线仍可能重新混乱

## 为什么现在做
因为当前项目已经进入长期维护和答辩支持阶段，如果没有协作底盘，后续 AI 难以持续接手。
